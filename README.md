# 校园失物招领系统

## 一、项目简介

校园失物招领系统是一个基于 Web 的校园服务平台功能模块，主要用于解决校园内物品丢失、拾取信息发布不及时、查找效率低、信息分散等问题。

用户可以在平台中发布失物信息或招领信息，查看他人发布的相关内容，并通过联系方式进行认领沟通。管理员可以对用户发布的信息进行审核和管理，保证平台内容真实、规范、安全。

本项目适用于高校校园场景，可作为校园综合服务平台中的一个功能模块，也可以独立部署使用。

---

## 二、项目背景

在校园生活中，学生经常会遇到校园卡、钥匙、书本、耳机、U 盘、雨伞、证件等物品丢失或拾取的情况。传统方式通常依靠班级群、表白墙、朋友圈或线下公告进行传播，存在信息分散、查找困难、重复发布、真实性难以确认等问题。

因此，本项目设计并实现一个统一的校园失物招领平台，帮助学生快速发布、搜索和查看失物招领信息，提高物品找回效率，优化校园服务体验。

## 三、项目功能

### 1. 用户注册与登录

- 用户可以通过账号注册平台账户
- 注册时填写基本信息
- 支持邮箱验证
- 注册后需要管理员审核
- 审核通过后用户可以正常登录
- 被封禁用户无法登录系统

### 2. 发布失物信息

用户可以发布自己丢失的物品信息，包括：

- 物品名称
- 物品分类
- 丢失地点
- 丢失时间
- 物品描述
- 联系方式
- 物品图片

发布后信息进入待审核状态，管理员审核通过后才会展示在前台页面。

### 3. 发布招领信息

用户拾取到他人物品后，可以发布招领信息，包括：

- 拾取物品名称
- 拾取地点
- 拾取时间
- 物品描述
- 联系方式
- 物品图片

管理员审核通过后，其他用户可以在平台查看该招领信息。

### 4. 失物招领列表展示

平台首页或失物招领页面展示已通过审核的信息，用户可以浏览：

- 最新失物信息
- 最新招领信息
- 物品名称
- 发布时间
- 发布人
- 物品状态
- 详情入口

### 5. 搜索功能

用户可以根据关键词搜索失物或招领信息，例如：

- 物品名称
- 丢失地点
- 拾取地点
- 分类
- 发布时间

方便用户快速找到相关信息。

### 6. 详情查看

用户点击信息后，可以查看完整详情，包括：

- 物品图片
- 详细描述
- 丢失或拾取时间
- 丢失或拾取地点
- 联系方式
- 发布人信息
- 当前状态

### 7. 管理员审核

管理员可以在后台查看所有待审核信息，并进行：

- 审核通过
- 审核拒绝
- 删除违规信息
- 查看发布详情
- 管理用户发布内容

审核机制可以避免虚假信息、广告信息和恶意发布，提高平台可信度。

### 8. 状态管理

失物招领信息支持不同状态，例如：

- 待审核
- 已通过
- 已拒绝
- 已找回
- 已归还
- 已删除

用户可以在物品找回后修改状态，避免无效信息长期展示。

## 六、核心业务流程

### 1. 用户注册流程

```text
用户填写注册信息
        ↓
发送邮箱验证码
        ↓
用户输入验证码
        ↓
后端校验验证码
        ↓
注册信息保存到数据库
        ↓
账号状态设为 pending
        ↓
等待管理员审核
        ↓
审核通过后可以登录
```

### 2. 发布失物招领流程

```text
用户登录系统
        ↓
填写失物或招领信息
        ↓
上传图片并提交
        ↓
信息进入 pending 状态
        ↓
管理员后台审核
        ↓
审核通过后展示到前台
```

### 3. 物品认领流程

```text
用户浏览或搜索信息
        ↓
进入详情页面
        ↓
查看联系方式
        ↓
联系发布人确认物品
        ↓
物品归还或找回
        ↓
修改信息状态
```

## 八、项目特点

### 1. 审核机制

所有注册用户和发布信息都需要管理员审核，提高平台安全性和信息真实性。

### 2. 邮箱验证

注册时通过邮箱验证码验证用户邮箱，减少无效注册和恶意注册。

### 3. 搜索便捷

用户可以通过关键词快速搜索失物和招领信息，提高找回效率。

### 4. 状态清晰

每条信息都有明确状态，方便用户判断物品是否已经找回或归还。

### 5. 校园场景适配

系统围绕校园生活设计，适合学生日常使用，界面简洁，操作方便。

---

## 九、安全设计

为了保证系统安全性，项目加入以下防护措施：

- 密码使用哈希加密保存
- 注册信息需要管理员审核
- 邮箱验证码设置有效期
- 验证码使用后立即失效
- 限制验证码发送频率
- 限制用户发布内容
- 管理员可删除违规信息
- 防止未登录用户发布信息
- 防止普通用户访问管理员页面
- 对用户输入进行基本校验和过滤

## 十一、运行方式

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置数据库

在 MySQL 中创建数据库：

```sql
CREATE DATABASE campus_service DEFAULT CHARSET utf8mb4;
```

修改项目中的数据库连接配置，例如：

```python
SQLALCHEMY_DATABASE_URI = "mysql+pymysql://用户名:密码@localhost/campus_service"
```

### 3. 配置邮箱

在项目根目录创建 `.env` 文件：

```env
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=465
MAIL_USERNAME=your_email@gmail.com
MAIL_PASSWORD=your_app_password
MAIL_SENDER=your_email@gmail.com
```

如果使用 163 邮箱：

```env
MAIL_SERVER=smtp.163.com
MAIL_PORT=465
MAIL_USERNAME=your_email@163.com
MAIL_PASSWORD=your_163_authorization_code
MAIL_SENDER=your_email@163.com
```

### 4. 启动项目

```bash
python app.py
```

浏览器访问：

```text
http://127.0.0.1:5000
```
